# Lark-Cli 消息发送规则
当用户提及"lark-alert"，在**命令开始执行**时，用lark-cli发送通知。

每条消息必须保证发出后才进行下一步，如果消息异常(HTTP 429 除外)，请对照lark-im skill或官方最新文档进行修正，最多重试3次，如果仍然失败，则跳过此次消息，进行下一步。

## 基础配置
- **身份**: Bot (`--as bot`)
- **目标群组**: `--chat-id oc_xxx`
- **消息格式**: Markdown (`--markdown`)

## 触发场景 & 消息标题

| 场景 | 标题格式 | 消息内容 | 示例 |
|------|---------|---------|------|
| 复杂任务开始 | `## 🚀 【具体任务名称】` | 任务详情 | `## 🚀 【实现背包管理系统】` |
| 复杂任务进度 | `## 📊 【具体任务名称 - 进度X/Y】` | 步骤进度 | `## 📊 【实现背包管理系统 - 进度1/5】` |
| 复杂任务完成 | `## ✅ 【具体任务名称】` | 完成总结 | `## ✅ 【实现背包管理系统】` |
| Plan 写/执行 | `## 📝 【Plan执行】` | 每步进度 | `## 📝【背包管理系统 - 进度1/5】` |
| Git Commit | `## 📦 【Git Commit】` | Commit摘要 | `## 📦 【背包管理系统 Commit】` |
| 任务中断/异常 | `## ⚠️ 【具体任务名称】` | 原因说明 | `## ⚠️ 【实现背包管理系统-异常】` |
| 需要用户操作 | `## 🎯 【需要用户操作】` | 操作指引 | `## 🎯 【bash waiting】` |
| 执行终端命令 | `## ⚠️ 【执行该命令的原因】` | 命令内容+原因说明 | `## ⚠️ 【删除旧文件】` |
| grill-me | `## 🔥【问题N：具体问题】` | 上次决策总结 + 新问题全文 | 见下方示例 |

## 触发条件

1. **复杂任务**: 经过思考后认为是复杂任务，则需要在开始、每步完成、结束时发消息
2. **Plan执行**: 写 plan 或执行 plan 时，每一步都要发消息
3. **Git Commit**: 每次 git commit 后，发送消息
4. **任务异常**: 任务中断/异常，或者需要用户操作时，发送消息
5. **执行终端命令**: 执行终端命令前，发送消息
6. **grill-me**: 在调用技能grill-me后，每次收到用户回答时，发送一条消息，包含：①上一个问题的决策总结 ②新提出的问题（含我的建议和选项）

## 消息格式规范

- 统一使用 **Markdown** 格式
- 复杂任务标题 = **具体任务名称**（带 emoji 和【】）
- 格式由 AI 自行判断（摘要/详情）

## 示例

### 复杂任务流程

```markdown
## 🚀 【实现背包管理系统】

这是一个复杂任务，包含以下步骤：
1. 分析需求
2. 设计架构
3. 实现核心功能
4. 测试验证
```

```markdown
## 📊 【实现背包管理系统 - 进度1/4】

已完成：分析需求
内容：...
```

```markdown
## ✅ 【实现背包管理系统】

任务完成！
内容：...
```

### Git Commit

```markdown
## 📦 【背包管理系统 Commit】

Commit: feat: add inventory system
内容：...
```

### grill-me

```markdown
## 🔥 【问题5→问题6】

### ✅ 问题5决策：内伤与AP递增交互
选A — 按实际消耗AP计算反噬

### 🔥 问题6：自动战斗AI与新AP机制
AI是否感知同技能AP递增？
- A. AI忽略递增（只按原始AP做预算）
- B. AI感知递增（优先交替使用不同技能）

我的建议：选B...
```
